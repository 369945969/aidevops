Redis Node.js 短信API 验证码

概述

基于Redis缓存的验证码存储架构,包含发送频率限制、过期管理、加密存储等核心设计要点,适用于短信验证码类功能开发。该架构方案从REQ-0007需求中提炼沉淀,经过AI架构师和人工审核双重验证后入库,已在28个相关需求场景中被引用复用,有效减少了重复设计工作,平均节省开发时间约40%。

核心设计要点

验证码过期管理 使用Redis的TTL机制,设置验证码5分钟自动过期,无需手动清理过期数据,确保验证码时效性
发送频率限制 基于Redis计数器实现每手机号每日5次发送上限,防止短信轰炸攻击,同时支持滑动窗口限流策略
加密存储设计 验证码在Redis中以SHA256哈希形式存储,避免明文存储带来的安全风险,校验时对用户输入进行哈希比对
降级容灾策略 Redis不可用时自动降级到内存缓存,短信服务商失败时自动切换备用渠道,确保核心验证功能不中断

代码示例: Redis验证码存储模式

sms-verify-store.js
Node.js 
class SmsVerifyStore {
  constructor(redisClient) {
    this.redis = redisClient;
    this.EXPIRE_SECONDS = 300; // 5分钟过期
    this.MAX_DAILY_COUNT = 5;  // 每日发送上限
  }

  async storeCode(phone, code) {
    // 频率限制检查
    const count = await this.redis.get(`sms:count:${phone}`);
    if (count >= this.MAX_DAILY_COUNT) {
      throw new Error('发送次数超限');
    }
    // SHA256哈希存储
    const hash = crypto.sha256(code);
    await this.redis.setex(
      `sms:code:${phone}`, this.EXPIRE_SECONDS, hash
    );
    // 递增每日计数
    await this.redis.incr(`sms:count:${phone}`);
  }
}
