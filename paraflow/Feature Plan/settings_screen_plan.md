# 系统设置
系统配置和管理页面,允许管理员配置AI Agent的自动化程度、人工审核节点、开发环境连接、团队成员权限等核心参数,控制整个平台的运行规则。

Layout Hierarchy:
- Header (Full-width):
  - Top Navigation Bar
- Content Container (Positioned below the header):
  - Left Settings Navigation
  - Main Settings Content Area

## Top Navigation Bar
- 产品Logo和名称
- 导航菜单:工作流看板、需求管理、知识库、系统设置(当前选中)
- 全局搜索框
- 新建需求按钮
- 通知中心图标
- 用户头像和下拉菜单

## Left Settings Navigation
- 设置分类:
  - 自动化配置(当前选中)
  - 审核规则
  - 开发环境
  - AI Agent配置
  - 团队成员
  - 通知设置
  - 集成配置
  - 安全与权限
  - 账单与订阅

## Main Settings Content Area
- 页面标题:
  - "自动化配置"标题
  - 保存更改按钮(主要操作,固定右上角)
  - 恢复默认设置链接
- 自动化级别设置:
  - 说明文字:"控制AI Agent的自动化程度,从保守到激进"
  - 滑动条选择器:
    - 保守模式:所有节点都需要人工审核
    - 平衡模式(推荐):关键节点人工审核
    - 激进模式:仅部署前人工审核
    - 完全自动(不推荐):全流程自动化,无人工介入
  - 当前选中模式的详细说明卡片
- 人工审核节点配置:
  - 说明文字:"选择哪些环节需要人工审核"
  - 审核节点开关列表:
    - 需求理解确认(默认:开启):
      - 开关按钮
      - 说明:AI分析需求后,需要人工确认是否理解正确
      - 影响:避免需求理解偏差导致后续工作无效
    - 架构方案审核(默认:开启):
      - 开关按钮
      - 说明:AI生成架构方案后,需要技术负责人批准
      - 影响:确保技术选型符合团队技术栈和长期规划
    - 代码合并审核(默认:开启):
      - 开关按钮
      - 子选项:仅核心模块需要审核 / 所有代码都需要审核
      - 核心模块定义输入框
      - 说明:AI开发完成后,需要人工审查代码质量
      - 影响:保证代码质量,防止安全漏洞
    - 测试结果确认(默认:关闭):
      - 开关按钮
      - 说明:自动化测试完成后,需要人工确认结果
      - 影响:增加一道质量保障,但会延长交付时间
    - 部署计划审核(默认:开启):
      - 开关按钮
      - 子选项:仅生产环境需要审核 / 所有环境都需要审核
      - 说明:AI生成部署计划后,需要人工批准上线
      - 影响:避免误操作导致生产事故
- AI行为偏好:
  - 说明文字:"调整AI决策时的偏好"
  - 技术选型偏好:
    - 倾向保守(使用成熟稳定的技术)
    - 中立(平衡新旧技术)
    - 倾向创新(优先考虑新技术)
  - 代码风格偏好:
    - 简洁优先
    - 注释详尽
    - 性能优先
    - 可读性优先
  - 测试覆盖率要求:
    - 滑动条:60% - 100%
    - 当前设置:80%(推荐)
- 超时与重试配置:
  - AI任务超时时间:
    - 需求分析超时(分钟):输入框,默认30
    - 架构设计超时(分钟):输入框,默认60
    - 代码开发超时(分钟):输入框,默认120
    - 测试执行超时(分钟):输入框,默认60
  - 失败重试策略:
    - 自动重试次数:下拉选择0-5次,默认2次
    - 重试间隔(分钟):输入框,默认5
    - 连续失败后的处理:
      - 自动通知人工介入
      - 暂停任务等待处理
      - 标记失败并跳过
- 保存提示:
  - 页面底部固定栏
  - "您有未保存的更改"提示(当有修改时显示)
  - 保存按钮
  - 取消按钮
