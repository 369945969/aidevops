# 代码审核
技术负责人审核AI生成代码的专用页面,展示代码diff、AI代码审查Agent的审查报告、安全漏洞扫描结果,支持逐文件审查和快速批准合并或要求修改。

Layout Hierarchy:
- Header (Full-width):
  - Top Navigation Bar
- Content Container (Positioned below the header):
  - Main Content Area with Three-Column Layout

## Top Navigation Bar
- 产品Logo和名称
- 导航菜单:工作流看板、需求管理、知识库、系统设置
- 全局搜索框
- 新建需求按钮
- 通知中心图标
- 用户头像和下拉菜单

## Main Content Area
- 面包屑导航:
  - 工作流看板 > 代码审核 > [任务ID]
- 审核概览卡片:
  - 任务ID和标题
  - 关联需求链接
  - 开发AI Agent名称
  - 代码提交时间
  - AI自动审查结果徽章:通过/需要修改/有风险
- 左侧文件树(文件导航):
  - 变更文件列表:
    - 新增文件(绿色标识)
    - 修改文件(黄色标识)
    - 删除文件(红色标识)
  - 文件路径展示
  - 变更行数统计(+XX -XX)
  - 点击文件切换中间区代码展示
  - 筛选器:全部/仅有问题的文件
- 中间代码展示区(主要内容):
  - 文件头部:
    - 当前文件路径
    - 文件类型图标
    - 快速导航:上一文件/下一文件按钮
  - Diff视图:
    - 并排对比模式:旧代码 | 新代码
    - 统一Diff模式切换
    - 语法高亮
    - 行号显示
    - 变更行高亮(绿色新增/红色删除)
  - AI审查标注:
    - 潜在问题行标记图标
    - 点击查看详细问题说明
    - 优化建议提示气泡
  - 人工批注功能:
    - 对任意行添加评论
    - 标记需要修改的代码段
    - @提及开发团队成员
- 右侧审查报告(辅助信息):
  - AI代码审查摘要:
    - 代码质量评分(0-100)
    - 复杂度分析结果
    - 测试覆盖率统计
    - 代码规范符合度
  - 问题列表:
    - 严重问题(红色):
      - 问题描述
      - 所在文件和行号
      - 点击定位到代码
    - 警告问题(黄色)
    - 优化建议(蓝色)
  - 安全扫描结果:
    - 发现的安全漏洞列表
    - CVE编号(如有)
    - 漏洞等级:高/中/低
    - 修复建议
  - 测试执行结果:
    - 单元测试通过率
    - 集成测试状态
    - 失败的测试用例链接
  - 审核操作区:
    - 批准合并按钮(主要操作,绿色):
      - 仅当无严重问题时可点击
    - 要求修改按钮(次要操作,黄色):
      - 输入修改意见表单
      - 选择需要修改的文件
      - 指定修改优先级
      - 提交后通知AI重新开发
    - 查看完整测试报告链接
    - 查看部署预览环境按钮(如已构建)
