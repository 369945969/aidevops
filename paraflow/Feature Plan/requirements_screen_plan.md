# 需求管理
需求列表页面,展示所有产品需求的概览信息,支持按状态、优先级、创建时间等维度筛选和排序,用户可以快速浏览需求进展并创建新需求。

Layout Hierarchy:
- Header (Full-width):
  - Top Navigation Bar
- Content Container (Positioned below the header):
  - Main Content Area

## Top Navigation Bar
- 产品Logo和名称
- 导航菜单:工作流看板、需求管理(当前选中)、知识库、系统设置
- 全局搜索框
- 新建需求按钮
- 通知中心图标
- 用户头像和下拉菜单

## Main Content Area
- 页面标题区:
  - "需求管理"标题
  - 新建需求按钮(主要操作)
  - 筛选器:全部 / 进行中 / 待审核 / 已完成 / 已归档
  - 排序选项:最新创建 / 优先级 / 更新时间
  - 搜索框:按需求标题或ID搜索
- 需求列表:
  - 需求卡片(多个):
    - 需求ID和标题
    - 需求描述摘要(前100字)
    - 当前状态标签:需求分析/架构设计/开发中/测试/已完成
    - 优先级标签:高/中/低
    - 创建者头像和名称
    - 创建时间和最后更新时间
    - 关联任务数量统计
    - 快速操作按钮:查看详情、编辑、归档
  - 空状态提示(当无需求时):
    - 插图
    - "还没有需求,点击上方按钮创建第一个需求"提示文字
    - 新建需求按钮
- 分页器:
  - 每页显示10条记录
  - 页码导航
  - 总记录数显示
